Proyecto 01 · Flutter · GCP · Firebase · Serverless

Armario Virtual

App móvil con motor de IA en la nube para gestión inteligente del vestuario personal

🐙 GitHub

¿Qué es?

Armario Virtual va más allá del simple almacenamiento de imágenes. Integra un motor de Inteligencia Artificial en la nube para analizar, categorizar y recomendar prendas automáticamente. El procesamiento de imágenes usa Dart Isolates para no bloquear la interfaz, garantizando una experiencia completamente fluida mientras la IA trabaja en segundo plano.

Funcionalidades principales

🧥
Gestión de Inventario con IA

Subida de fotos con recorte automático de bordes transparentes y optimización de imagen. API externa integrada para eliminación de fondos dejando las prendas limpias visualmente.

🏷️
Auto-Tagging automático

Identifica el tipo de prenda ("Camiseta", "Vaqueros") y sus colores predominantes en RGB de forma automática. Confianza mínima del 70% garantizada por Vision API.

🛍️
Smart Shopping visual

Usa los tags generados por la IA para buscar y sugerir productos similares en la web en tiempo real mediante Google Custom Search API.

🌐
Traducción automática

Traduce las etiquetas detectadas del inglés al español con Translation API, filtrando términos irrelevantes como "textil" o "producto" mediante lista de exclusión personalizada.

👗
Probador Virtual (Fitting Room)

Interfaz Drag & Drop para componer outfits combinando parte superior, inferior y calzado. Validación de categorías esenciales antes de guardar.

🔐
Seguridad avanzada

Firebase Auth con Google Sign-In, Firebase App Check con Play Integrity contra tráfico abusivo y Secret Manager en GCP para gestión segura de claves API.

Arquitectura Cloud — GCP (europe-west1)

Cloud Functions (Python Gen 2)Backend serverless — lógica de IA completamente desacoplada del cliente
Vision APIEtiquetado automático con confianza >70% y extracción de colores RGB dominantes
Translation APITraducción EN→ES con filtro de exclusión personalizado por dominio
Custom Search APIMotor de búsqueda visual por tags para recomendación de productos similares
Cloud FirestoreBD NoSQL optimizada para lecturas en tiempo real por colección de usuario
Cloud StorageAlmacenamiento multimedia por usuario: garment_images/{userId}/
Firebase Auth + App CheckAutenticación con Google Sign-In y protección Play Integrity
Secret ManagerGestión segura de claves API sin exposición en código fuente

Stack técnico

Proyecto 02 · Unity · C# · Spring Boot · Azure · Kubernetes

Minigames Platform

Plataforma de entretenimiento completa: cliente Unity con 6 juegos y backend de microservicios

🐙 API Repo 🎮 Game Repo

¿Qué es?

Plataforma de entretenimiento completa compuesta por un cliente desarrollado en Unity con menú principal interactivo (configuración de volumen, pantalla completa y calidad gráfica) y una robusta arquitectura backend de microservicios para gestión integral de jugadores, autenticación y auditoría. Desarrollado como TFG del Grado Superior DAM.

Catálogo de juegos

🏓 Pong Glow — mejor de 3 🐦 Flappy Bird — supervivencia ⛄ Snowball War — multijugador local 🦠 Jimmy vs Covid — esquiva obstáculos 🚀 Space Invaders — con jefe final 🦊 Foxy Bros — plataformas tipo Mario

Arquitectura de microservicios (Backend)

API Gateway (puerto 8080)Punto de entrada unificado para todas las peticiones HTTP con enrutamiento dinámico
Auth ServiceRegistro y autenticación de usuarios — comunica eventos vía RabbitMQ al servicio de auditoría
User Operations ServiceBúsqueda de jugadores por username y progresión de nivel
Audit ServiceRegistro asíncrono de eventos de registro y login recibidos por RabbitMQ
Exception HandlerControlador global de excepciones con respuestas HTTP estandarizadas

Infraestructura & DevOps

Infraestructura completamente dockerizada y desplegada en Azure con Kubernetes. Pipeline CI/CD automatizado con GitHub Actions: flujos de limpieza, instalación y publicación de imágenes Docker en Docker Hub. Base de datos SQL Server con dos schemas segregados: uno para usuarios y otro para auditoría.

Stack técnico

Proyecto 03 · Spring Boot · Node.js · Azure · Kubernetes

Gestión Genérica

Prácticas ViewNext — Ecosistema de 6 microservicios con CI/CD completo y despliegue en Azure

🐙 GitHub

¿Qué es?

Aplicación empresarial diseñada para gestionar operaciones genéricas, construida íntegramente bajo un paradigma de microservicios altamente escalable. Backend en Java Spring Boot y frontend en Node.js, con un entorno de despliegue continuo completamente automatizado. Desarrollada durante las prácticas en ViewNext.

Ecosistema de 6 microservicios

API GatewayEnrutamiento dinámico de todas las peticiones al microservicio correspondiente
User CRUDOperaciones completas de creación, lectura, actualización y eliminación de usuarios
Security ServiceImplementación de Spring Security para autenticación y autorización centralizada
Audit ServiceTrazabilidad de registros y accesos recibidos de forma asíncrona vía RabbitMQ
Exception ServiceManejo centralizado de excepciones personalizadas con respuestas HTTP estandarizadas
Discovery Service (Eureka)Service discovery para balanceo de carga interno entre microservicios

Comunicación asíncrona

Implementación de RabbitMQ como broker de mensajería para el envío de datos de registro y login hacia el servicio de auditoría. Esta arquitectura evita cuellos de botella en la autenticación y garantiza la trazabilidad sin penalizar el rendimiento de los servicios principales.

Infraestructura & DevOps

Entorno completamente dockerizado y desplegado en Azure con Kubernetes. El ciclo de vida del software está automatizado con pipelines de Jenkins y GitHub Actions para integración y despliegue continuo (CI/CD). El proyecto está respaldado con documentación funcional completa en wikis de GitHub.
